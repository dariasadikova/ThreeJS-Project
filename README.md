# Three.js Interactive Scene and Model Loader

## Описание проекта

Данный проект представляет собой веб-приложение на базе Three.js, реализующее интерактивную 3D-сцену с возможностью управления камерой, освещением и объектами, а также загрузки пользовательских 3D-моделей. Проект выполнен в рамках учебного задания и разделён на две логические части.

Приложение собирается и запускается с использованием Vite, использует ES-модули и может быть развернуто как локально, так и на хостинге.

**Протестировать работу приложения можно здесь:**  https://threejspro-samike.amvera.io

## Часть 1. Базовая 3D-сцена

В первой части реализована интерактивная сцена со следующими возможностями:

* Навигация камеры с помощью OrbitControls
* Включённые тени
* Использование текстуры-изображения на одном из объектов сцены
* Не менее трёх источников света:

  * AmbientLight
  * DirectionalLight
  * PointLight
* Использование минимум двух разных типов освещения
* Наличие четырёх объектов на сцене
* Использование BufferGeometry для двух объектов:

  * Плоскость (PlaneGeometry)
  * Треугольная пирамида, созданная вручную через BufferGeometry
* Управление параметрами сцены через стандартные HTML-элементы ввода:

  * Интенсивность DirectionalLight
  * Интенсивность AmbientLight
  * Цвет PointLight
  * Цвет пирамиды

---

## Часть 2. Работа с пользовательскими моделями

Во второй части добавлена работа с пользовательскими 3D-моделями:

* Загрузка моделей в форматах `.glb` и `.gltf`:

  * через `<input type="file">`
  * через перетаскивание файлов (drag-and-drop)
* Выбор модели кликом по объекту на сцене
* Управление выбранной моделью с помощью TransformControls:

  * перемещение
  * вращение
  * масштабирование
* Поддержка горячих клавиш:

  * W — перемещение
  * E — вращение
  * R — масштабирование
  * Esc — снятие выделения
* Числовые поля для точного задания трансформаций:

  * Position (X, Y, Z)
  * Rotation (в градусах)
  * Scale (X, Y, Z)
* Двусторонняя синхронизация:

  * изменение через мышь обновляет числовые поля
  * изменение числовых полей влияет на модель

## Локальный запуск проекта

### Требования

* Node.js версии 18 или выше
* npm

### Установка и запуск

```bash
npm install
npm run dev
```

После этого приложение будет доступно по адресу, указанному в консоли Vite.

---

## Сборка проекта

```bash
npm run build
```

Результат сборки будет помещён в папку `dist`.

---

## Поддерживаемые форматы моделей

* `.glb` (рекомендуется)
* `.gltf` (возможна загрузка вместе с `.bin` и текстурами)

---

## Примечания

* Для тестирования загрузки моделей рекомендуется использовать формат `.glb`.
* Проект ориентирован на учебные цели и демонстрирует базовые и расширенные возможности Three.js.
